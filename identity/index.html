01

Filosofia de Design

O lfc-admin-gui é uma ferramenta de poder. Administra identidades, permissões e acesso. O design deve refletir isso: preciso, confiável, sem ruído. Estética "secure terminal" — técnica sem ser fria.

🎯
Clareza antes de tudo

Interfaces administrativas tratam de dados críticos. Cada elemento deve ser imediatamente legível. Hierarquia visual nítida, sem ambiguidades sobre o que é destrutivo ou irreversível.

🔒
Segurança visível

O sistema lida com JWT, permissões e controle de acesso. O design deve comunicar segurança e controle — não com alarmes visuais, mas com precisão, consistência e feedback confiável.

🌿
Verde como identidade

A paleta de verdes do ecossistema LFC — do floresta ao lima — carrega tanto a natureza orgânica quanto a acuidade técnica. Não é "sucesso": é identidade de marca.

⚡
Densidade inteligente

Administradores trabalham com dados densos: tabelas, permissões, rotas. O layout deve suportar densidade sem sufocamento — espaço branco é respiração, não desperdício.

🔧
Feedback imediato

Cada ação tem consequência. Micro-interações confirmam, alertam e guiam. Estados de loading, erro e sucesso são obrigatórios — o usuário nunca deve se perguntar "funcionou?".

♿
Acessibilidade técnica

Contraste mínimo WCAG AA, focus visível, ARIA labels corretos. Não é checklist: é respeito ao usuário que opera o sistema em qualquer condição de trabalho.

08

Motion & Animação

Animações devem confirmar ações, não decorar. Use ease-out para elementos que aparecem, ease-in para os que saem. Nunca animar mais de 2 elementos simultâneos sem propósito claro.

Fade
200ms · ease-out · opacity
Tooltips, modais, dropdowns — elementos que aparecem sobre o conteúdo.
Slide
300ms · cubic-bezier(0.16,1,0.3,1)
Sidebars, painéis laterais, drawers de detalhes.
Scale
200ms · cubic-bezier(0.34,1.56,0.64,1)
Badges de status, confirmações, ícones de sucesso/erro.
Regras de Motion
✅ Máximo 200ms para interações de hover e focus
✅ Loading states sempre após 300ms de espera
✅ Stagger máximo de 50ms entre elementos em lista
✅ Reducir/eliminar animações com prefers-reduced-motion
❌ Não animar layout (width/height) — usar transform
❌ Não usar animações em mensagens de erro críticas
09

Voz & Tom

O sistema fala com administradores técnicos. Direto, preciso, sem suavizações desnecessárias. Feedback de ação deve ser concreto: qual recurso, qual operação, qual resultado.

✓ Fazer
✓"Sistema desativado. Rotas continuam registradas."
✓"Permissão perm:Users.Delete atribuída a admin@lfc.com.br"
✓"Sessão invalidada. tokenVersion incrementado para 13."
✓"Erro 403: sem permissão para esta operação."
✓"12 usuários afetados por esta role."
✕ Não fazer
✕"Ops! Algo deu errado. Tente novamente mais tarde."
✕"Tem certeza que quer fazer isso?" (sem contexto)
✕"Sucesso! ✨ Operação realizada com êxito!"
✕"Você não pode fazer isso agora."
✕"Carregando... por favor aguarde."
